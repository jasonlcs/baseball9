# Retro Baseball

以 HTML5 Canvas 製作的復古桌上棒球小遊戲。  
核心目標是把「機台感」做出來：節奏、手感、視覺回饋、以及可理解的棒球規則。

## 設計理念

1. 機台感優先  
以固定尺寸球場、誇張但可預期的球路與打擊回饋，重現街機式棒球節奏。

2. 可讀性優先  
畫面上明確顯示壘包、佔壘狀態、壘上跑者娃娃頭、S/B/O 資訊與落點，讓玩家快速理解局面。

3. 規則簡化但不失真  
保留棒球核心流程（好壞球、三振、保送、進壘、三出局換邊、九局制），讓遊戲保持易玩且有策略性。

4. 守備有情境  
守備站位會依壘上狀態與出局數調整（如趨前守備、雙殺深度、短打警戒），不是固定站樁。

5. 擊球手感強化  
擊中後有短暫慢速（bullet-time）與震動感，讓每次擊球更有「打到球」的存在感。

## 遊戲規則

### 基本局制
- 九局制，含上下半局。
- 每半局三出局換邊。
- 九局結束後結算比數（目前不做延長賽流程）。

### 好球 / 壞球 / 出局
- 揮空或進好球帶未揮棒會累積好球。
- 四壞保送會觸發強迫進壘。
- 三好球判定三振出局。

### 擊球結果（簡化判定）
- 落地距離較遠：可能形成一壘安打 / 二壘安打 / 全壘打。
- 飛球被接住：出局。
- 打得太短：可能形成出局。
- 界外球會增加好球（最多到兩好球）。

### 佔壘與得分
- 壘包狀態以一壘、二壘、三壘布林值追蹤。
- 畫面上會同步顯示：
  - 主球場壘包高亮（佔壘時）
  - 壘上跑者娃娃頭（大小與守備娃娃頭一致）
  - 左下角小壘包 HUD

### 守備站位邏輯（特色）
- 壘上有人時，守備會做對應 hold 與中線收縮。
- 少於 2 出局且有特定壘況時，內野會趨前守備。
- 一壘有人且可抓雙殺時，二游站位會調整為雙殺深度。
- 短打警戒情境下，角落內野（1B/3B）會明顯趨前。

## 操作說明

### 你的角色
- 上半局：你是投手（防守方）
- 下半局：你是打者（進攻方）

### 操作鍵位
- 投球：在可投球狀態下，點擊球場（以上半局為主）
- 揮棒：`Space` 或滑鼠左鍵（下半局）
- 投球型態切換（上半局）：
  - `1` FAST
  - `2` CURVE
  - `3` CHANGE

### 手機投球機制
- 上半局（你是投手）：
  - 先用下方 `FAST / CURVE / CHANGE` 按鈕選擇球種
  - 在球場上按住並拖曳準星
  - 放開手指後出手投球
- 下半局（你是打者）：
  - 點擊球場揮棒
